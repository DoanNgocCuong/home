So sánh v1 vs v2:

|Điểm|v1 (Part A / Part B)|v2 (7 Phases)|
|---|---|---|
|**Cấu trúc**|2 phần: Live + Distilled|7 phases theo đúng dòng chảy problem solving|
|**Flow**|Viết log → Đúc kết sau|Vấn đề → Nguyên nhân → Giải pháp → Thử nghiệm → Triển khai → Kiểm tra → Đúc kết|
|**Giả thuyết**|Gộp trong Investigation Log|Tách riêng: **Hypothesis table** (2.1) + **Kiểm chứng từng cái** (2.2)|
|**MECE**|Có Solution Space nhưng chung chung|Thêm tiêu chí đánh giá: Impact / Urgency / Effort / **Leverage** / Risk|
|**Verification**|Checklist ngắn|Phase riêng (6) với commands + checklist + metrics before/after|
|**Đúc kết**|Part B riêng|Phase 7 tích hợp: Mistakes + Best Practices + Action Items + **Quick Reference Card**|
|**Report**|Không có|Có sẵn **REPORT format** cuối file — copy-paste gửi stakeholder|

Triết lý áp dụng từ file Problem Solving của bạn:

- **First Principles** (Tầng 2): Phase 2 bắt buộc bóc tách giả thuyết trước, kiểm chứng từng cái — không nhảy thẳng vào fix
- **MECE** (Tầng 2+3): Phase 3 yêu cầu liệt kê TẤT CẢ options, đánh giá theo **Leverage** (làm 1 dùng N lần)
- **Kaizen** (Tầng 7): Phase 7 là phần đúc kết — "Không đo lường không cải tiến"
- **"Nghi ngờ giải pháp của mình"**: Ghi rõ trong Phase 2 header
