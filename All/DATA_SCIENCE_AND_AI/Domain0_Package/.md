Đây là **tóm tắt lý thuyết cốt lõi** về `pyproject.toml` và việc  **đóng gói (packaging) Python project** :

---

## 1. `pyproject.toml` là gì?

* File cấu hình chuẩn của Python project (theo PEP 518).
* Dùng để mô tả:
  * **Cách build** (build-system: setuptools, poetry, hatch...).
  * **Metadata** : tên, phiên bản, mô tả, file README, yêu cầu Python.
  * **Dependencies** : các thư viện cần cài.
  * **Rules** cho setuptools khi tìm package (include/exclude).

---

## 2. Đóng gói (Packaging) là gì?

* **Đóng gói = biến project thành 1 package có thể cài đặt bằng `pip install`** .
* Kết quả tạo ra:
  * File `.whl` (wheel) → cài nhanh.
  * File `.tar.gz` (source).

👉 Giúp code của bạn trở thành **sản phẩm** giống như `numpy`, `fastapi`.

---

## 3. Quá trình đóng gói & cài đặt

1. **pip đọc `pyproject.toml`** → biết dùng công cụ build nào.
2. **pip tải dependencies** (từ PyPI hoặc cache).
3. **Build project** thành `.whl`.
4. **Cài đặt vào environment** → bạn có thể `import` ở bất kỳ đâu.

---

## 4. Cache pip

* Pip lưu tạm các `.whl` trong thư mục cache (VD: `C:\Users\User\AppData\Local\Temp\pip-...`).
* Mục đích: tăng tốc cài đặt, không phải tải lại mỗi lần.

---

## 5. Cách cài đặt lại nhanh nhất

* **Dev mode** (dành cho đang code):

  ```bash
  pip install -e .
  ```

  (tự động update khi sửa code).
* **Force reinstall** :

```bash
  pip install --force-reinstall .
```

* **Uninstall rồi cài lại** :

```bash
  pip uninstall agent-core -y
  pip install .
```

---

## 6. Cách share package

* **Nội bộ** :
* Share file `.whl` → cài bằng `pip install file.whl`.
* Hoặc share GitHub repo → `pip install git+https://github.com/...`.
* **Công khai** :
* Upload lên **PyPI** bằng `twine upload dist/*`.
* Người khác chỉ cần `pip install agent-core`.

---

📌  **Cốt lõi** :

* `pyproject.toml` = hợp đồng để build + cài project.
* Đóng gói = biến code thành 1 package `.whl`/`.tar.gz`.
* Share = gửi file `.whl`, repo Git, hoặc publish lên PyPI.
