# SAI LẦM KHI KHÔNG FIX CỨNG VERSION CỦA LIBRARY - 

> Luồng lesson 12-03-2026

> Bug rơi vào đầu trên dev khi qua mới update và redeploy. Nhưng mình đâu sửa gì đến luồng chính mà báo lỗi 

**`langfuse==4.0.0` vừa release ngày 10/03/2026** — đúng 2 ngày trước. Trong `requirements.txt` đang ghi `langfuse>=3.3.4`, nên khi rebuild Docker image tối qua, pip tự kéo v4.0.0 về. Đó là lý do "tự động đổi từ v3 sang v4" — không ai đổi, pip resolve version mới nhất thỏa `>=3.3.4`.

**Fix ngay — pin v3 trong requirements.txt:**

```
langfuse>=3.3.4,<4.0.0
```

Version v3 cuối cùng là `3.14.5`. Nếu muốn pin cứng:

```
langfuse==3.14.5
```

Sau đó rebuild container là chạy lại bình thường.

**Bài học:** luôn pin upper bound cho dependencies trong production để tránh breaking changes khi major version mới release. Format an toàn nhất là `>=3.3.4,<4.0.0` (cho phép patch/minor updates nhưng chặn major).
