# Worktree vs Branch - Hiểu Đúng Để Dùng Đúng

**Mục đích:** Phân biệt rõ ràng giữa **Worktree** và **Branch**, khi nào dùng cái nào trong dự án photo-to-lesson.

---

## 1. So Sánh Core Concept

| Khía cạnh | Branch | Worktree |
|-----------|--------|----------|
| **Định nghĩa** | Con trỏ tới commit trong Git history | Thư mục working directory độc lập |
| **Số lượng** | 1 working directory duy nhất | Nhiều thư mục song song |
| **Khi switch** | Phải `stash` hoặc mất WIP | Không mất gì, cd vào là làm việc |
| **Dùng cho** | Quản lý version code | Chạy song song nhiều task |

### Visual Explanation

```
BRANCH (con trỏ trong Git):
main    ─────► [commit A] ─────► [commit B] ─────► [commit C]  ← HEAD
dev     ─────► [commit D] ─────► [commit E]

WORKTREE (thư mục thực tế):
/repo/main         → Working dir 1 (port 30001)
/repo/.worktree/v2 → Working dir 2 (port 30002)
/repo/.worktree/v3 → Working dir 3 (debug)
```

---

## 2. Khi Nào Dùng Cái Nào?

### ✅ Dùng **Branch** khi:

- Quản lý version code (feature/hotfix/release)
- Cần merge code vào main
- Team collab qua PR
- Lightweight, không cần chạy song song

### ✅ Dùng **Worktree** khi:

- **Deploy test riêng** (port khác) mà không stop production
- **Debug long-running** process mà không interrupt dev
- **Review PR** trên local
- **Experiment** code mà không ảnh hưởng dev hiện tại
- **CI/CD testing** cần isolated environment

## 3. Ứng Dụng Trong Dự Án Photo-to-Lesson

### Bài toán thực tế:

```
Tình huống: Cần test Langfuse @observe nhưng sợ break production đang chạy
→ Dùng Worktree để deploy môi trường test riêng (port 30002)
→ Code ổn định → Merge vào main
→ Xong → Dọn dẹp worktree
```

### Architecture:

```
┌─────────────────────────────────────────────────────────────────┐
│                        GIT REPOSITORY                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  Branch: main ───────────────────────────────────────────► :30001 │
│   │         (production đang chạy)                             │
│   │                                                            │
│   └──► Branch: worktree-deploy-test-30002 ─────────────► :30002│
│            (test @observe, có thể break)                       │
│                                                                  │
│  Branch: dev ──────────────────────────────────────────────► backup │
│           (giữ code cũ Redis+Worker)                          │
└─────────────────────────────────────────────────────────────────┘
```

### Use Cases Cụ Thể:

| Use Case | Dùng | Lý do |
|----------|-------|-------|
| Deploy test port 30002 | **Worktree** | Cần port riêng, không ảnh hưởng :30001 |
| Thử nghiệm @observe | **Worktree** | Code chưa stable, có thể break |
| Giữ code Redis+Worker | **Branch: dev** | Backup để roll back nếu cần |
| Merge khi ổn định | **Branch: main** | Production deployment |
| Hotfix bug gấp | **Branch mới** | Quick fix, merge nhanh |

## 4. Workflow Thực Tế Trong Dự Án

### Scenario: Test @observe Langfuse

```bash
# 1. Tạo worktree từ main (production đang chạy ổn định)
git worktree add .claude/worktrees/deploy-test-30002 -b worktree-deploy-test-30002

# 2. Trong worktree: Thử nghiệm @observe
cd .claude/worktrees/deploy-test-30002
# ... modify code, test ...

# 3. Commit vào worktree branch khi ổn
git add .
git commit -m "refactor: use @observe instead of Redis worker"

# 4. Merge vào main (production)
git checkout main
git merge worktree-deploy-test-30002

# 5. Dọn dẹp worktree
git worktree remove .claude/worktrees/deploy-test-30002
git branch -d worktree-deploy-test-30002
```

### Giữ Code Cũ Làm Backup:

```bash
# Tạo dev branch từ main TRƯỚC KHI merge
git checkout main
git checkout -b dev  # Giữ code cũ (Redis+Worker)
git merge worktree-deploy-test-30002
# Bây giờ dev = main = @observe
# Nhưng có thể roll back nếu cần
```

## 5. So Sánh Với Cách Khác

### Cách 1: Chỉ dùng Branch (không worktree)

```bash
# Problem: Phải stop production để test
git checkout -b test-branch
# → Code ở đây, production bị interrupt
# → Không deploy được song song
```

### Cách 2: Dùng Worktree (recommended)

```bash
# Solution: Deploy test song song, không ảnh hưởng production
git worktree add .claude/worktrees/test -b test-branch
# → Production vẫn chạy ở port 30001
# → Test chạy ở port 30002
# → Hoàn toàn độc lập
```

## 6. Best Practices Cho Dự Án

### Recommended Setup:

```
/repo (main)
├── .claude/worktrees/
│   ├── deploy-test-30002/     # Test features mới
│   └── review-pr-xxx/         # Review PR
└── branches:
    ├── main                    # Production
    ├── dev                     # Backup/Rollback
    └── feature-xxx            # Features đang dev
```

### Quy Tắc:

| Rule | Lý do |
|------|-------|
| **Worktree cho test deploy** | Isolated environment, port riêng |
| **Branch cho code management** | Merge, PR, version control |
| **Dev branch để backup** | Roll back khi có vấn đề |
| **Xóa worktree sau khi merge** | Tránh confusion, code lỗi thời |

## 7. Commands Thường Dùng

```bash
# Worktree
git worktree list                           # Xem tất cả worktree
git worktree add <path> -b <branch>        # Tạo worktree mới
git worktree remove <path>                  # Xóa worktree

# Branch
git branch                                 # Liệt kê branches
git checkout -b <branch>                   # Tạo + switch branch
git merge <branch>                         # Merge vào branch hiện tại

# Cleanup
git worktree prune                          # Dọn dẹp worktree refs
```

---

## 8. Troubleshooting

### Worktree conflict

```bash
# Xem worktree đang có
git worktree list

# Force remove nếu cần
git worktree remove --force <path>
```

### Branch đang ở worktree

```bash
# Không xóa được branch vì đang ở worktree
# → Di chuyển sang branch khác trước
git checkout main
git branch -D <branch>
```
