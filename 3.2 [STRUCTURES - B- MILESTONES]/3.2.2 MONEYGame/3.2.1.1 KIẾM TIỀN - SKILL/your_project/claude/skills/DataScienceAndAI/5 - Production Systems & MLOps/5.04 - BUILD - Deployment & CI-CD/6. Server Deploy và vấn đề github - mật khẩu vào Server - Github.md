## 2.4 SSH Key 

GitHub không còn hỗ trợ password authentication qua HTTPS nữa. Cần dùng **Personal Access Token (PAT)** hoặc **SSH key**.

**Cách nhanh nhất — dùng PAT:**

1. Tạo token tại: **GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic)** → Generate new token, tick scope `repo`.
    
2. Push với token:
    

```bash
git push https://doanngoccuong-stepupeducation:<PAT_TOKEN>@github.com/IsProjectX/robot-mem0-oss.git
```

3. Để không phải nhập lại mỗi lần:

```bash
git remote set-url origin https://doanngoccuong-stepupeducation:<PAT_TOKEN>@github.com/IsProjectX/robot-mem0-oss.git
```

**Hoặc chuyển sang SSH (recommended lâu dài):**

```bash
# Tạo SSH key nếu chưa có
ssh-keygen -t ed25519 -C "your-email@stepup.edu.vn"

# Copy public key
cat ~/.ssh/id_ed25519.pub
# → Paste vào GitHub Settings → SSH and GPG keys → New SSH key

# Đổi remote sang SSH
git remote set-url origin git@github.com:IsProjectX/robot-mem0-oss.git
git push
```

Thêm lỗi `vscode-git-*.sock` kia là do VS Code credential helper bị disconnect — không ảnh hưởng gì nếu dùng PAT hoặc SSH trực tiếp. Nếu muốn tắt nó:


### Chưa hiểu, thế về sau có 2 account 1 cái git cá nhân, 1 cái git công ty thì đẩy code kiểu gì để nhanh chuyển đổi giữa 2 tài khoản 

> Nhân sự kiện: Ông anh đổi hết quyền truy cập từ tk cá nhân của mình thành quyền tk công ty 

> 1. Nếu muốn máy tính mình ssh được vào server thì gửi ssh key của mình vào server. 
> 2. Nếu muốn server truy cập được vào github thì gửi ssh key của server vào github (**GitHub account personal** → Settings → SSH and GPG keys → **New SSH key** → paste public key vào → Save)

Đúng rồi, cơ chế SSH key pair hoạt động như sau:

Mỗi máy tạo ra 1 cặp key: **private key** (giữ bí mật trên máy đó) + **public key** (gửi cho nơi muốn truy cập vào).

**Máy mình → Server:** Máy mình giữ private key, gửi public key lên server (vào `~/.ssh/authorized_keys`). Khi SSH vào, server dùng public key verify rằng máy mình đang giữ đúng private key → cho vào.

**Server → GitHub:** Server giữ private key (`id_doanngoccuong_personal`), gửi public key lên GitHub (SSH keys settings). Khi git push/pull, GitHub dùng public key verify rằng server đang giữ đúng private key → cho truy cập repo.

Nói gọn: **public key luôn đặt ở "cửa" mình muốn vào, private key luôn giữ ở máy muốn đi qua cửa đó.** Như ổ khóa (public) và chìa khóa (private) vậy — đặt ổ ở đâu thì mở cửa ở đó.

Vì server shared nhiều người dùng, nên SSH config mới cần nhiều alias (`github.com-doanngoccuong-personal`, `github.com-thanhdt24`...) để mỗi người dùng đúng private key của mình khi push lên đúng GitHub account.

```bash
# 1. Tạo 2 SSH key (nhấn Enter 3 lần cho mỗi cái để skip passphrase)
ssh-keygen -t ed25519 -C "cuongmkmtpgoldfinch@gmail.com" -f ~/.ssh/id_doanngoccuong_personal
ssh-keygen -t ed25519 -C "cuongdn.rd@stepup.edu.vn" -f ~/.ssh/id_doanngoccuong_stepup

# 2. Tạo SSH config
cat > ~/.ssh/config << 'EOF'
# Cuong DN - Account cá nhân
Host github-doanngoccuong-personal
    HostName github.com
    User git
    IdentityFile ~/.ssh/id_doanngoccuong_personal

# Cuong DN - Account công ty StepUp
Host github-doanngoccuong-stepup
    HostName github.com
    User git
    IdentityFile ~/.ssh/id_doanngoccuong_stepup
EOF

chmod 600 ~/.ssh/config

# 3. In ra 2 public key để paste lên GitHub
echo "========== KEY CÁ NHÂN (paste vào github account personal) =========="
cat ~/.ssh/id_doanngoccuong_personal.pub
echo ""
echo "========== KEY CÔNG TY (paste vào github account stepup) =========="
cat ~/.ssh/id_doanngoccuong_stepup.pub

# 4. Git config theo thư mục
git config --global user.name "Cuong DN"
git config --global user.email "cuongmkmtpgoldfinch@gmail.com"

cat > ~/.gitconfig-stepup << 'EOF'
[user]
    name = Cuong DN
    email = cuongdn.rd@stepup.edu.vn
EOF

git config --global includeIf.gitdir:~/stepup/.path ~/.gitconfig-stepup
```

Chạy xong sẽ in ra 2 public key. Copy từng cái paste vào đúng GitHub account:

- **Key cá nhân** → github.com (account personal) → Settings → SSH and GPG keys → New SSH key
- **Key công ty** → github.com (account stepup) → Settings → SSH and GPG keys → New SSH key

Sau đó test:

```bash
ssh -T github-doanngoccuong-personal 
ssh -T github-doanngoccuong-stepup
```

# Checklist Debug SSH + Git Remote trên Shared Server

| Step                             | Lệnh                             | Kết quả đúng                                                          | Lỗi thường gặp                                                                     | Cách fix                                                                              |
| -------------------------------- | -------------------------------- | --------------------------------------------------------------------- | ---------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------- |
| **1. Check remote URL**          | `git remote -v`                  | `git@<SSH-ALIAS>:<org>/<repo>.git` — chỉ **1 dấu `:`**                | 2 dấu `:` như `git@github.com:alias:org/repo.git` hoặc alias không khớp SSH config | `git remote set-url origin git@<SSH-ALIAS>:<org>/<repo>.git`                          |
| **2. Check SSH config**          | `cat ~/.ssh/config`              | Có `Host` match đúng alias trong remote URL, ghi nhận `IdentityFile`  | Không có entry cho alias, hoặc alias bị typo                                       | Thêm entry vào `~/.ssh/config` với đúng `Host`, `HostName github.com`, `IdentityFile` |
| **3. Check key file tồn tại**    | `ls -la ~/.ssh/<identity_file>*` | Có private key (permission `600`) + public key (`.pub`)               | File không tồn tại hoặc sai permission                                             | `ssh-keygen -t ed25519 -C "email" -f ~/.ssh/<key_name>` rồi `chmod 600`               |
| **4. Test SSH connection**       | `ssh -T git@<SSH-ALIAS>`         | `Hi <username>! You've successfully authenticated...`                 | `Permission denied (publickey)`                                                    | Public key chưa add lên GitHub → xem Step 5                                           |
| **5. Add public key lên GitHub** | `cat ~/.ssh/<identity_file>.pub` | Copy output → GitHub Settings → SSH Keys → New SSH key → Paste → Save | Key add sai account hoặc paste thiếu                                               | Đảm bảo add đúng GitHub account tương ứng                                             |
| **6. Push / Pull**               | `git push origin main`           | Push thành công                                                       | Vẫn fail → quay lại Step 1 kiểm tra từ đầu                                         | —                                                                                     |

## Nguyên tắc cốt lõi

| Khái niệm             | Giải thích                                                                                        |
| --------------------- | ------------------------------------------------------------------------------------------------- |
| **SSH Key Pair**      | Private key giữ ở máy muốn truy cập, public key đặt ở nơi muốn vào (GitHub, server)               |
| **SSH Alias**         | `Host` trong `~/.ssh/config` — dùng thay `github.com` để phân biệt nhiều account trên cùng server |
| **Remote URL format** | `git@<Host-alias>:<owner>/<repo>.git` — **KHÔNG** có `github.com` nếu đã dùng alias               |
