## 1. SSH vào Server

### Lý thuyết

SSH (Secure Shell) cho phép kết nối terminal từ xa vào server một cách bảo mật. Trên server dùng chung (shared server), mọi lệnh cần `sudo` để có quyền root.

Có 2 cách xác thực: **password** (đơn giản nhưng kém an toàn) hoặc **SSH key pair** (recommended).

### Thực hành

**Cách 1: SSH bằng password**

```bash
ssh ubuntu@103.253.20.30
# Nhập mật khẩu khi được hỏi
```

**Cách 2: SSH qua VSCode (recommended — code trực tiếp trên server)**

1. Tạo hoặc chỉnh sửa file `C:\Users\User\.ssh\config`:

```plaintext
Host my-server
    HostName 103.253.20.30
    User ubuntu
    Port 22
    IdentityFile C:\Users\User\.ssh\id_rsa
```

> **Lưu ý:** Nếu đăng nhập bằng mật khẩu, có thể bỏ dòng `IdentityFile`.

2. Mở VSCode → `Ctrl+Shift+P` → **Remote-SSH: Connect to Host** → chọn `my-server`.

### Công cụ quản lý file trên Server

| Công Cụ | Nền Tảng | Chức Năng | Giao Thức | Đối Tượng |
|---------|----------|-----------|-----------|-----------|
| **WinSCP** | Windows | Duyệt file, kéo thả, chỉnh sửa trực tiếp | SFTP, SCP | Người dùng Windows muốn giao diện giống Explorer |
| **FileZilla** | Windows, macOS, Linux | Truyền file với giao diện đơn giản | FTP, SFTP | Quản lý file qua giao diện dễ làm quen |
| **MobaXterm** | Windows | SSH client tích hợp quản lý file | SSH, SFTP | Dev cần SSH & quản lý file trong 1 app |
| **VSCode** | Windows, macOS, Linux | IDE + Remote-SSH để duyệt/chỉnh sửa file | SSH | Dev muốn chỉnh sửa file trực tiếp |
| **Cockpit** | Web-based | Quản lý server qua trình duyệt | Web | Quản lý server qua giao diện web |

## 2. Các lệnh hữu ích trên Server

### Lý thuyết

Khi đã SSH vào server, cần nắm vài lệnh cơ bản: kiểm tra container, xem cấu trúc thư mục, dọn dẹp disk.

### Thực hành

```bash
# Xác định container chạy trên port cụ thể
sudo docker ps | grep 25039

# Xem cấu trúc thư mục dạng cây
sudo apt install tree -y    # cài tree nếu chưa có
tree -L 2

# Dọn dẹp Docker — xóa containers, images, volumes không dùng
docker system prune -af --volumes
```

## 3. Git Authentication — SSH Key vs PAT

### Lý thuyết

GitHub đã **ngừng hỗ trợ password authentication qua HTTPS**. Chỉ còn 2 cách:

| Phương thức | Ưu điểm | Nhược điểm | Khi nào dùng |
|-------------|---------|------------|-------------|
| **SSH Key** (recommended) | Setup 1 lần, không cần nhập lại. An toàn. | Cần setup ban đầu | Dùng lâu dài, đặc biệt trên server |
| **Personal Access Token (PAT)** | Nhanh, không cần tạo key | Token có hạn, cần quản lý | Fix nhanh, CI/CD, hoặc khi chưa setup SSH |

**Nguyên lý SSH Key Pair:**

> **Public key** luôn đặt ở "cửa" mình muốn vào, **private key** luôn giữ ở máy muốn đi qua cửa đó.
> Như ổ khóa (public) và chìa khóa (private) — đặt ổ ở đâu thì mở cửa ở đó.

- **Máy mình → Server:** Public key lên server (`~/.ssh/authorized_keys`), private key giữ ở máy.
- **Server → GitHub:** Public key lên GitHub (SSH Keys settings), private key giữ ở server.

### Thực hành — Setup SSH Key

```bash
# 1. Kiểm tra SSH key đã có chưa
ls -al ~/.ssh

# 2. Tạo SSH key mới (nếu chưa có)
ssh-keygen -t ed25519 -C "your-email@example.com"
# Hoặc dùng RSA:
ssh-keygen -t rsa -b 4096 -C "your-email@example.com"

# 3. Lấy public key → copy toàn bộ output
cat ~/.ssh/id_ed25519.pub

# 4. Add public key lên GitHub:
#    GitHub → Settings → SSH and GPG keys → New SSH Key → Paste → Save

# 5. Test kết nối
ssh -T git@github.com
# Kết quả đúng: "Hi your-username! You've successfully authenticated..."

# 6. Nếu gặp "Permission denied (publickey)":
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519
```

### Thực hành — Setup PAT (cách nhanh)

```bash
# 1. Tạo token: GitHub → Settings → Developer settings → Personal access tokens
#    → Tokens (classic) → Generate new token → tick scope "repo"

# 2. Clone hoặc push với token:
git clone https://DoanNgocCuong:YOUR_TOKEN@github.com/DoanNgocCuong/your-repo.git

# 3. Hoặc set remote để không nhập lại:
git remote set-url origin https://DoanNgocCuong:YOUR_TOKEN@github.com/DoanNgocCuong/your-repo.git
```

> **Lưu ý:** Không chia sẻ token cho người khác. Nếu lỡ lộ → GitHub → revoke token ngay.

## 4. Multi-Account Git trên cùng Server

### Lý thuyết

Khi nhiều người dùng chung 1 server, hoặc 1 người có nhiều GitHub account (cá nhân + công ty), cần **SSH alias** để phân biệt account nào dùng key nào.

**Cơ chế:** File `~/.ssh/config` tạo alias (ví dụ `github-personal`, `github-company`). Khi git remote dùng alias thay vì `github.com`, SSH tự chọn đúng private key.

### Thực hành — Ví dụ thực tế: 2 account (cá nhân + công ty StepUp)

```bash
# 1. Tạo 2 SSH key riêng biệt
ssh-keygen -t ed25519 -C "cuongmkmtpgoldfinch@gmail.com" -f ~/.ssh/id_doanngoccuong_personal
ssh-keygen -t ed25519 -C "cuongdn.rd@stepup.edu.vn" -f ~/.ssh/id_doanngoccuong_stepup

# 2. Tạo SSH config
cat > ~/.ssh/config << 'EOF'
# Cuong DN - Account cá nhân
Host github-doanngoccuong-personal
    HostName github.com
    User git
    IdentityFile ~/.ssh/id_doanngoccuong_personal

# Cuong DN - Account công ty StepUp
Host github-doanngoccuong-stepup
    HostName github.com
    User git
    IdentityFile ~/.ssh/id_doanngoccuong_stepup
EOF

chmod 600 ~/.ssh/config

# 3. In ra 2 public key để paste lên GitHub
echo "=== KEY CÁ NHÂN (paste vào github account personal) ==="
cat ~/.ssh/id_doanngoccuong_personal.pub

echo "=== KEY CÔNG TY (paste vào github account stepup) ==="
cat ~/.ssh/id_doanngoccuong_stepup.pub

# 4. Test kết nối từng account
ssh -T github-doanngoccuong-personal
# → Hi DoanNgocCuong! You've successfully authenticated...

ssh -T github-doanngoccuong-stepup
# → Hi doanngoccuong-stepupeducation! You've successfully authenticated...
```

### Git config theo thư mục (auto chọn email đúng)

```bash
# Global config (mặc định = account cá nhân)
git config --global user.name "Cuong DN"
git config --global user.email "cuongmkmtpgoldfinch@gmail.com"

# Config riêng cho thư mục công ty
cat > ~/.gitconfig-stepup << 'EOF'
[user]
    name = Cuong DN
    email = cuongdn.rd@stepup.edu.vn
EOF

git config --global includeIf.gitdir:~/stepup/.path ~/.gitconfig-stepup
```

> **Kết quả:** Mọi repo trong `~/stepup/` tự dùng email công ty. Repo khác dùng email cá nhân.

#### Câu hỏi: Best practices nên là github-doanngoccuong-... hay là github.com-doanngoccuong-...

Nên dùng `github.com-doanngoccuong-...`

Lý do chính: khi clone repo, bạn chỉ cần thay đổi tối thiểu từ URL gốc.

**So sánh khi clone:**

```bash
# URL gốc từ GitHub
git clone git@github.com:doanngoccuong/repo.git

# Với github.com-... → chỉ thay phần host, pattern rõ ràng
git clone git@github.com-doanngoccuong-personal:doanngoccuong/repo.git

# Với github-... → mất cái ".com", dễ nhầm là host khác
git clone git@github-doanngoccuong-personal:doanngoccuong/repo.git
```

**Lý do phụ:**

Giữ `github.com` prefix giúp nhận diện ngay đây là alias của `github.com`, không phải GitHub Enterprise hay self-hosted server nào khác. Nếu sau này thêm GitLab, pattern vẫn nhất quán: `gitlab.com-doanngoccuong-...`.

Config nên là:

```bash
Host github.com-doanngoccuong-personal
    HostName github.com
    User git
    IdentityFile ~/.ssh/id_doanngoccuong_personal

Host github.com-doanngoccuong-stepup
    HostName github.com
    User git
    IdentityFile ~/.ssh/id_doanngoccuong_stepup
```

## 5. Pull Code — Clone repo về Server

### Lý thuyết

Clone repo bằng **SSH URL** (không dùng HTTPS) để tránh lỗi authentication.

**Format SSH URL:**
- 1 account: `git@github.com:<owner>/<repo>.git`
- Multi-account (dùng alias): `git@<SSH-ALIAS>:<owner>/<repo>.git`

> **Quan trọng:** Chỉ có **1 dấu `:`** giữa alias và owner. Sai thường gặp: `git@github.com:alias:org/repo.git` (2 dấu `:`) → lỗi "not a valid repository name".

### Thực hành

```bash
# Clone bằng SSH (1 account)
git clone git@github.com:DoanNgocCuong/MiniProd_Web4_ContentEngFlow_StepUpE_T102024.git

# Clone bằng SSH alias (multi-account)
git clone git@github-doanngoccuong-personal:DoanNgocCuong/robot-parents-lesson-builder.git
git clone git@github-doanngoccuong-stepup:IsProjectX/robot-mem0-oss.git

# Clone repo GitLab (dùng PAT)
git clone https://oauth2:YOUR_TOKEN@gitlab.com/platform-rag/robot-ai-workflow.git

# Clone repo GitLab (dùng SSH)
git clone git@gitlab.com:platform-rag/robot-ai-workflow.git
```

### Bug thường gặp: Clone HTTPS bị "Invalid username or password"

```
remote: Invalid username or password.
fatal: Authentication failed for 'https://github.com/DoanNgocCuong/...'
```

**Nguyên nhân:** GitHub không cho dùng username/password nữa.
**Fix:** Chuyển sang SSH hoặc dùng PAT thay cho password.

## 6. Push Code — Đẩy code lên GitHub

### Lý thuyết

Trước khi push, cần đảm bảo:
1. **Git config** đúng tên & email (để commit hiển thị đúng author)
2. **Remote URL** đúng format (SSH hoặc HTTPS+PAT)
3. **Quyền truy cập** — account đang dùng phải có quyền push lên repo đó

### Thực hành

```bash
# Set tên và email cho repo hiện tại
git config --local user.name "Doan Ngoc Cuong"
git config --local user.email "cuongmkmtpgoldfinch@gmail.com"

# Kiểm tra config
git config --local --list

# Kiểm tra remote URL
git remote -v

# Đổi remote sang SSH (nếu đang dùng HTTPS)
git remote set-url origin git@github-doanngoccuong-stepup:IsProjectX/robot-mem0-oss.git

# Push
git push origin main
```

### Bug thường gặp: "Permission denied to thanhdt24"

```
ERROR: Permission to DoanNgocCuong/repo.git denied to thanhdt24.
fatal: Could not read from remote repository.
```

**Nguyên nhân:** Server đang dùng SSH key của `thanhdt24` thay vì key của bạn.
**Fix:** Dùng SSH alias trong remote URL:

```bash
# SAI (dùng github.com → SSH chọn key mặc định, có thể là key người khác)
git remote set-url origin git@github.com:DoanNgocCuong/repo.git

# ĐÚNG (dùng alias → SSH chọn đúng key của bạn)
git remote set-url origin git@github-doanngoccuong-personal:DoanNgocCuong/repo.git
```

### Bug thường gặp: "not a valid repository name"

```bash
# SAI — 2 dấu ":"
git remote set-url origin git@github.com:doanngoccuong-personal:DoanNgocCuong/repo.git

# ĐÚNG — alias thay cho github.com, chỉ 1 dấu ":"
git remote set-url origin git@doanngoccuong-personal:DoanNgocCuong/repo.git
```

## 7. Checklist Debug SSH + Git Remote

| Step | Lệnh | Kết quả đúng | Lỗi thường gặp | Cách fix |
|------|-------|-------------|----------------|---------|
| **1. Check remote URL** | `git remote -v` | `git@<ALIAS>:<org>/<repo>.git` — chỉ 1 dấu `:` | 2 dấu `:` hoặc alias không khớp SSH config | `git remote set-url origin git@<ALIAS>:<org>/<repo>.git` |
| **2. Check SSH config** | `cat ~/.ssh/config` | Có `Host` match đúng alias, có `IdentityFile` | Không có entry hoặc alias bị typo | Thêm entry với đúng `Host`, `HostName github.com`, `IdentityFile` |
| **3. Check key file** | `ls -la ~/.ssh/<key>*` | Có private key (permission `600`) + `.pub` | File không tồn tại hoặc sai permission | `ssh-keygen -t ed25519 -f ~/.ssh/<key>` rồi `chmod 600` |
| **4. Test SSH** | `ssh -T git@<ALIAS>` | `Hi <username>! You've successfully authenticated...` | `Permission denied (publickey)` | Public key chưa add lên GitHub → Step 5 |
| **5. Add key lên GitHub** | `cat ~/.ssh/<key>.pub` | Copy → GitHub Settings → SSH Keys → New → Paste → Save | Key add sai account hoặc paste thiếu | Đảm bảo add đúng GitHub account tương ứng |
| **6. Push / Pull** | `git push origin main` | Push thành công | Vẫn fail → quay lại Step 1 | — |

## 8. Tóm tắt — Nguyên tắc cốt lõi

| Khái niệm | Giải thích |
|-----------|-----------|
| **SSH Key Pair** | Private key giữ ở máy muốn truy cập, public key đặt ở nơi muốn vào (GitHub, server) |
| **SSH Alias** | `Host` trong `~/.ssh/config` — dùng thay `github.com` để phân biệt nhiều account trên cùng server |
| **Remote URL format** | `git@<Host-alias>:<owner>/<repo>.git` — **KHÔNG** có `github.com` nếu đã dùng alias |
| **PAT** | Thay thế password cho HTTPS. Tạo tại GitHub → Settings → Developer settings → Tokens |
| **git config --local** | Set tên & email riêng cho từng repo. Ưu tiên hơn `--global` |
